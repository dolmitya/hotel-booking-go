# Hotel Booking System

Сервис для управления бронированием номеров в отеле.

Проект реализован на Go с использованием Gin и демонстрирует построение backend-приложения с нуля: от работы с базой данных до метрик и API.

## Функциональность

Система позволяет:
- управлять гостями (создание, редактирование, получение);
- управлять номерами (CRUD-операции);
- создавать и изменять бронирования;
- искать доступные номера на заданный период;
- отслеживать загрузку отеля.

## Архитектура

Проект построен по классической трехслойной архитектуре:

- **Handler** — обработка HTTP-запросов  
- **Service** — бизнес-логика и основные проверки  
- **Repository** — работа с базой данных через SQL  

Такое разделение упрощает поддержку, тестирование и масштабирование системы.

## Работа с базой данных

Для доступа к данным используется `pgx` и нативные SQL-запросы.

### Почему не ORM:
- полный контроль над SQL-запросами;
- предсказуемое поведение;
- явная работа со схемой и индексами.

### Альтернатива:
Можно было использовать ORM, что уменьшило бы количество кода, но усложнило бы контроль над запросами и структурой доступа к данным.

## Бизнес-логика

В сервисном слое реализованы основные проверки:
- существование гостя и номера;
- проверка вместимости номера;
- отсутствие пересечений бронирований;
- корректность временных интервалов.

## События (Kafka)

Реализована отправка событий в Kafka при:
- создании и изменении гостей;
- операциях с номерами;
- создании и обновлении бронирований.

Это позволяет интегрировать систему с другими сервисами.

## Метрики

Добавлены кастомные метрики:
- количество созданных и обновленных бронирований;
- текущая загрузка отеля (occupancy rate).

Метрики могут использоваться в системах мониторинга (реализовано с Prometheus и Grafana).

## API документация

API задокументировано с использованием Swagger (OpenAPI).
